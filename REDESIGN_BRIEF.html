1. 產品背景與使用者

產品定位

台股 + 美股混合投資組合追蹤儀表板，特色是內建「Serenity 供應鏈透鏡」— 針對 AI/半導體高波動選股的策略頁面。個人使用工具，非 SaaS。

核心使用者

零售投資者，熟悉 Bloomberg / TradingView 語彙，同時關注長線持股與短線題材股。使用場景：早晨開盤前檢視、盤後結算、週末重新配置。

目前痛點

  • 資訊密度不夠 —— 桌面版空間浪費，關鍵數字（總損益 / 報酬率）視覺權重與市值相同
  • 圖表與表格風格未統一，深色底色偏中性，缺乏「金融終端」的專業感
  • Serenity 頁卡片太長，滾動負擔重；配置比例的條狀圖辨識度低
  • 行動裝置未優化 —— sidebar 佔滿螢幕
  • 沒有清楚的資訊層級（primary / secondary metric 不分）

2. 設計原則與方向

主張「Reuters Workstation × Linear」

資訊密度接近 Bloomberg / Reuters 終端，但視覺俐落度向 Linear、Notion、Vercel 這類現代 SaaS 靠攏。專業但不擁擠。

氛圍Dark-first · Data-forward

純深色主題（不做 light mode），數字用等寬字級 tabular figures，圖表色板飽和但不刺眼。避免 glassmorphism / neumorphism。

Principle 1 · Numbers are the hero
金額、報酬率、市值 —— 這些數字要用最大字級、tabular-nums、對齊右邊。所有次要資訊（icon、label）都要弱化。
Principle 2 · 綠漲、紅跌（沿用西方慣例，維持既有）
台股習慣紅漲綠跌，但既有 codebase 用 green/red = profit/loss 的西方慣例。統一維持既有：綠漲、紅跌，避免使用者混淆；色階用偏青綠（#10b981）而非螢光綠。
Principle 3 · 8pt grid · zero magic numbers
所有 spacing 用 4 / 8 / 12 / 16 / 24 / 32 / 48 / 64。所有 border-radius 用 4 / 6 / 10 / 16。禁止 7px、13px、23px 這類 magic number。
Principle 4 · Progressive disclosure
Dashboard 一眼看完；細節（技術指標、Serenity checklist）點開才顯示。避免所有東西同時佔螢幕。

10. 給 Claude Design 的三個小技巧

  • 先建 DS 再開 Project：DS 是可複用的資產，直接在 Prototype 裡建的樣式無法跨檔案共用
  • 上傳現有 HTML 檔比截圖有用：從截圖它只能猜結構，從 HTML 它可以精確保留 id / class 契約，之後 Claude Code 對回程式碼會順很多
  • 先 Wireframe 敲 layout，再 Prototype 上樣式：如果對五頁 layout 還不確定，先用 Wireframe 模板產灰階線框圖，快速迭代 3–5 次敲定 layout 骨架後再走 Prototype 上高保真樣式，會省很多重繪時間
