P1-1移除 CORS 代理單點故障

問題：所有 Yahoo Finance 呼叫都經過 api.allorigins.win，第三方公用代理隨時可能被限速或停用，導致整個 app 抓不到資料。
方案：加入多來源 fallback（Yahoo → FinMind → 台灣證交所公開資料）並允許使用者自訂 proxy endpoint。
影響檔案：app.js（fetchPrice / fetchPriceFallback / fetchHistory）
▸ Claude Code Prompt
請重構 app.js 裡的 fetchPrice、fetchPriceFallback、fetchHistory 三個函式，達成以下目標：

1. 抽出一個 PROVIDERS 陣列，依序嘗試：
   (a) 使用者自訂 proxy（從 localStorage 讀取 key: "customProxy"）
   (b) api.allorigins.win（現有）
   (c) corsproxy.io
   (d) 台股：若 market === "TW"，嘗試證交所 https://openapi.twse.com.tw/v1/exchangeReport/STOCK_DAY_AVG_ALL 或 FinMind 免費 API
2. 每個 provider 加 5 秒 timeout（用 AbortController）
3. 任一 provider 回傳有效資料就 return，全部失敗才回傳 zero-object
4. 在 console 印出實際成功的 provider 名稱，方便除錯
5. 新增一個 UI 元件（在 sidebar-footer 下方）讓使用者填入自訂 proxy URL，存 localStorage
6. 保留原本函式簽章，避免其他呼叫端要改

改完後在檔案頂端加註解說明 provider 順序與 fallback 邏輯。

P1-2修補 XSS 注入風險

問題：news-title、stock.name、source 等來自外部/使用者輸入的字串，透過 innerHTML 直接插入 DOM。惡意新聞標題可注入 <script> 或 onerror= 攻擊。
方案：建立 escapeHtml() 工具、所有動態字串一律 escape；連結 href 白名單 https/http。
影響檔案：app.js（fetchNews / renderHoldingsTable / renderManageTable / addStock）
▸ Claude Code Prompt
請為 app.js 加上完整 XSS 防護：

1. 在檔案頂端加一個 utility：
   function escapeHtml(str) {
     if (str == null) return "";
     return String(str).replace(/[&<>"'`=\/]/g, s => ({
       "&":"&","<":"<",">":">",
       "\"":""","'":"'","`":"`","=":"=","/":"/"
     }[s]));
   }
2. 找出所有用 innerHTML 塞外部資料的地方（fetchNews、renderHoldingsTable、renderManageTable、renderPriceChart 的 label、showToast），全部改用 escapeHtml() 包住 stock.name、stock.symbol、title、source。
3. fetchNews 裡的 link 屬性：只接受 http:// 或 https:// 開頭，其他一律替換成 "#"。
4. addStock 收到的 symbol / name 也要在存進 portfolio 前做 trim + 長度上限（symbol ≤ 10 字元、name ≤ 40 字元）。
5. 完成後在 README 或程式頂端註明「所有外部資料需經 escapeHtml() 處理」。

方案：抓即時匯率，統一換算成 TWD 顯示；並在 UI 標示「以 TWD 計」與當日匯率。
影響檔案：app.js（新增 fetchFxRate / refreshAll / renderHoldingsTable），index.html（summary-cards 加註）
▸ Claude Code Prompt
請為 app.js 加入貨幣換算：

1. 新增 fetchFxRate() 函式，抓 USD→TWD 匯率（可用 Yahoo Finance 的 "TWD=X" 符號透過既有 CORS proxy）
2. refreshAll() 開頭先 await fetchFxRate()，把匯率存到全域 state.usdTwd
3. 計算 totalValue / totalCost 時，若 stock.market === "US"，把該筆金額乘以 state.usdTwd 換算成 TWD 再加總
4. renderHoldingsTable() 每列的「市值」欄位一律以 TWD 顯示；額外增加一欄「原幣值」顯示 stock.market 的原始金額（US 顯示 USD、TW 顯示 TWD）
5. 在 sidebar-footer 或 dashboard header 顯示：「當前 USD/TWD: xx.xx（{時間} 更新）」
6. 若 fetchFxRate 失敗，使用 fallback 32.5 並在 UI 用黃色警示標示「匯率為估計值」
7. 記得更新 index.html 的表格 header（新增「原幣值」欄）以及 colspan 數字

P2-2價格快取 + TTL

問題：priceCache 物件宣告了卻從未真正被讀取；且沒有過期機制。頻繁 refresh 會浪費 API 配額。
方案：實作 60 秒 TTL 記憶體快取；同時把當日收盤價存進 localStorage 作為離線 fallback。
影響檔案：app.js（fetchPrice / refreshAll）
▸ Claude Code Prompt
請重寫 app.js 的 priceCache 機制：

1. 把 priceCache 改為 { "TW:2330": { data: {...}, ts: 1720000000000 } } 結構
2. fetchPrice() 進入時先檢查 cache，若 (Date.now() - ts) < 60_000 直接回傳 data，不打 API
3. 抓成功後同時寫入 localStorage key: "priceCache"，並在檔案啟動時載入舊 cache 作為 offline fallback
4. 所有 API 都失敗時，若 localStorage 有 24 小時內的 cache，回傳並在 UI 標示「離線資料 · {幾小時前}」
5. 新增手動清 cache 按鈕（放在 dashboard 更新按鈕旁邊，icon: trash-alt）
6. refreshAll 收到全部 zero-object 時要 toast 「無法連線，顯示為快取資料」而不是 "數據已更新"
